# Prompt COMMAND_TEST

## 頁面目的
建立 Test Command 頁面，用於手動驗證單一 command 的執行結果。

## 生成要求
- 頁面應聚焦測試流程
- 需有輸入 payload 與輸出 log 區
- 適合工程測試與除錯

## 內容區塊
- 頁面標題：Test Command
- Test Runner 區塊
  - command selector
  - payload input
  - execute button
- Execution Log 區塊
  - request summary
  - mock response
  - success / error 狀態

## 風格限制
- 不要多餘裝飾
- 讓輸入與 log 可快速閱讀
